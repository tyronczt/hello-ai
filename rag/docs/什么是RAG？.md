# 什么是RAG？

**RAG (Retrieval-Augmented Generation，检索增强生成)** 是一种将强大的**信息检索 (Information Retrieval, IR)** 技术与**生成式大语言模型 (LLM)** 相结合的框架。**检索**指的是检索外部知识库，**增强生成**指的是将检索到的知识送给大语言模型以此来优化大模型的生成结果，使得大模型在生成更精确、更贴合上下文答案的同时，也能有效减少产生误导性信息的可能。

> 💡 **一句话总结**：RAG 就是让 LLM 学会了“开卷考试”，它既能利用自己学到的知识，也能随时查阅外部资料，主要也是为了解决存在的一些局限性。


RAG（Retrieval-Augmented Generation，检索增强生成）是一种优化大语言模型（LLM）的技术框架。它在生成回答前，先从外部知识库（如企业文档、数据库）中检索相关信息，再结合这些信息由LLM生成回答。该技术能有效减少AI幻觉、保证信息即时性，且无需重新训练模型。

RAG 的核心优势与流程：
工作原理： 用户提问 -> 检索相关文档 -> 将文档与问题发给LLM -> LLM生成准确回答。
解决幻觉： 相比直接向LLM提问，RAG让AI“有据可依”，显著降低回答的不准确性。
私有数据接入： 允许企业连接本地知识库，让LLM基于内部特定数据作答。
高效率与低成本： 无需昂贵的训练成本，只需更新外部知识库即可让模型掌握最新信息。

RAG常用于智能问答机器人、企业知识库检索和实时文档分析等场景。

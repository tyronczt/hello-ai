1. RAG 与传统微调的本质区别

微调通过更新大模型内部参数来学习新知识，将知识"固化"在模型权重中；而 RAG（Retrieval-Augmented Generation）不修改模型参数，而是通过动态检索外部知识库，在推理时实时注入最新、真实的信息。这种方式有效缓解了模型因训练数据截止导致的知识滞后问题，并显著降低**幻觉（Hallucination）**风险。

2. 为什么 RAG 能缓解大模型的幻觉问题？

RAG 的生成内容严格基于检索到的真实文档片段，而非依赖模型内部记忆。在生成过程中，LLM 的注意力机制会聚焦于相关上下文，从而将输出约束在可验证的事实范围内，大幅减少无依据的虚构内容。

3. 如何定义 RAG 中的"相关文档"？

相关文档是指在给定用户查询（Query）下，从知识库中通过语义相似度（如向量检索）或关键词匹配（如 BM25）筛选出的 Top-K 最相关文本片段。其"相关性"通常由检索模型打分决定。

4. RAG 中主流检索器有哪些？各自优势是什么？

- 稀疏检索（如 BM25）：基于词频和逆文档频率，擅长精确关键词匹配，计算高效，但缺乏语义泛化能力。
- 稠密检索（如 DPR、Sentence-BERT）：将文本映射为稠密向量，支持语义层面的相似性搜索，但依赖高质量嵌入模型和算力。
- 混合检索：结合稀疏与稠密方法（如 RRF 融合），兼顾关键词精准性和语义泛化能力，提升整体召回与准确率。

5. 知识库文档分块有哪些常用策略？

常见分块方式包括：
- 固定大小分块：实现简单、效率高，但可能割裂语义边界。
- 按语义分割：利用 NLP 工具（如句子边界检测、主题分割）保持上下文完整性。
- 递归/层次分块：先粗粒度分块，再对长段落递归细分，兼顾不同粒度需求。

此外还有滑动窗口（带重叠）、结构感知分块（按标题/章节）等策略。

6. 为什么需要对向量进行归一化？

归一化（如 L2 归一化）使向量仅保留方向信息，消除长度差异对相似度计算的影响。在使用余弦相似度时，归一化后点积即等于余弦值，确保语义距离衡量更准确。

7. 生成阶段如何融合检索结果？

主要有两种方式：
- 拼接输入（Concatenation）：将查询与检索到的多个文档拼接成单一 prompt 输入 LLM，例如 [CLS] Query [SEP] Doc1 [SEP] Doc2 ...。
- 中间层注入（如 Fusion-in-Decoder）：在 LLM 解码过程中，将每个文档独立编码并通过交叉注意力融合，避免上下文过长问题。

8. 如何避免 RAG 生成无关或错误内容？

- 设置检索置信度阈值，低相关性结果不参与生成。
- 在 prompt 中明确约束："仅根据以下文档回答，若信息不足请说明无法回答"。
- 引入 Self-RAG 等机制，让模型自主判断是否需要引用检索结果，或拒绝生成。

9. 为什么需要重排序（Re-ranking）？常见方法有哪些？

初检（如向量检索）返回的 Top-K 结果可能存在噪声或排序不准，需进一步精排：
- 交叉编码器（Cross-Encoder）：联合编码 Query 和 Doc，输出精细相关性分数（如 bge-reranker）。
- LLM 重排：用大模型对候选文档打分或排序，利用其更强的语义理解能力。

10. RAG 与微调如何结合？

- 两阶段训练：先微调检索器适配领域语料，再固定检索器微调生成器以更好利用检索结果。
- 端到端联合训练：如 RA-DIT 等方法，同步优化检索器与生成器，实现整体性能提升。

11. 如何评估 RAG 生成结果的质量？

- 事实准确性：与标准答案或权威来源对比。
- 引用精确度：生成内容是否忠实于引用文档，有无曲解。
- 自动指标：ROUGE、BLEU、BERTScore 等衡量文本相似性；也可结合人工评估。

12. 如何优化检索的召回率（Recall）？

- Query 扩展：使用同义词、LLM 改写或 Step-Back 提问生成多角度查询。
- HyDE（Hypothetical Document Embeddings）：让 LLM 生成假设答案，以其向量作为查询进行检索。
- 优化分块策略：采用重叠分块、多粒度索引等，减少关键信息遗漏。

13. RAG 如何处理多文档中的冲突信息？

- 提示词控制：要求 LLM 总结共识点，并明确标注分歧来源。
- 权威性加权：根据文档元数据（如来源类型、发布时间）赋予不同权重。
- 多视角输出：直接呈现不同观点，并说明存在冲突。

14. 如何解决"检索偏好"问题？

当检索结果质量差却强制生成时，易导致错误。解决方案包括：
- 引入检索质量评估模块，过滤低置信度结果。
- 设计回退机制：如返回"根据现有资料无法回答"。
- 采用迭代检索或 Re-rank 机制，提升结果可靠性。

15. 如何优化长文档的检索效果？

- Small-to-Big 检索：先检索小片段，再扩展至其所属完整章节或文档。
- 层次化检索：先定位相关章节，再在章节内细粒度检索。
- 图结构增强：构建知识图谱，通过实体关系链接分散的文档片段。

16. HyDE 的原理是什么？

HyDE（Hypothetical Document Embeddings）让 LLM 根据用户查询生成一个假设性答案，将其作为"伪文档"编码为向量，再用该向量去检索真实知识库。这种方法能弥合用户提问与文档表述之间的语义鸿沟，提升检索相关性。

17. 什么是迭代检索（Iterative Retrieval）？

迭代检索指多轮检索-生成循环：首轮检索结果输入 LLM 生成初步答案，再以该答案或中间推理作为新查询进行二次检索，直至满足终止条件（如答案收敛）。适用于复杂推理或多跳问答场景。

18. Self-RAG 的核心创新点是什么？

Self-RAG 引入可学习的检索决策机制，模型在生成过程中能自主判断是否需要检索（Retrieve-on-Demand），并通过特殊 token 控制检索与生成流程，实现更灵活、高效的 RAG 推理。

19. RAG 如何适配实时更新的知识库？

- 检索器后端使用支持增量更新的索引系统（如 Elasticsearch、Pinecone）。
- 生成器无需重新训练，但需监控新数据是否引起分布偏移，必要时更新嵌入模型或提示模板。

20. 用户问"2025年诺贝尔奖获得者"，但知识库只更新到2024年，RAG 应如何应对？

- 检索器返回截至 2024 年的最新获奖信息。
- 生成器明确回应："截至 2024 年，最新诺贝尔奖获得者为 XXX；2025 年结果尚未公布。"
- 可在系统层面加入时间敏感性检测，对涉及未来事件的问题自动触发免责声明。
